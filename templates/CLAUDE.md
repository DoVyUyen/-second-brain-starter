# Second Brain — <your name> (<role> @ <company>)

## Session Start (always do this first)
1. Read brain/index.md
2. Read last 5 entries of brain/log.md
3. Read work/active/ for any active project

> Tip: if you installed the SessionStart hook (see the starter repo), steps
> 1–3 happen automatically in every Claude Code session — in any project.

## During Session
- New decision made → save to decisions/ immediately (filename: YYYY-MM-DD-short-title.md)
- New knowledge → save to brain/topics/
- All notes use [[wikilinks]] to reference other files

## End of Session (when I say "wrap up")
1. Update relevant work/active/ file
2. Append to brain/log.md:
   `## [date] wrap-up | [what was done] | next: [next task]`

## Rules
- Never load more than 3 files per query
- Decisions are immutable once written — never edit, only supersede
- Never modify raw/ files (if that folder is added later)
- Keep topics/ focused: one concept per file, not one project per file
- **Team knowledge goes to your team's shared knowledge base, not here.** If
  you learn something about the product or business that teammates would
  benefit from, put it in the shared wiki/docs. The brain is for personal
  working state only.
